=head3 Vertical Dot Chart w/ Border and Labels

  my $tc = Term::Chart->new(
      {   orientation => 'vertical',
          style       => 'dot',
          border      => 1,
          tip         => "\x{29BF}",
      }
  );

  my @values = 0 .. 8;
  push @values, reverse 0 .. 7;

  for my $n (@values)
  {
      my %datum = (
          value => $n,
          label => $n,
      );
      $tc->add_value( \%datum );
  }

  print "$tc\n";

=head2 DESCRIPTION

This module offers you a convenient way to create bar & dot charts and for
your command line programs.

The charts can include ANSI color markup and Unicode drawing characters.

=head2 METHODS

=head3 new

The constructor accepts a hash ref with the following optional parameters.

=over

=item bar

The character used for the bars. The default is U+2585 for
horizontal, and U+2589 for vertical.

=item tip

The last character on each bar, at the tip. The default is U+2585 for
horizontal, and U+2589 for vertical.

=item height

The row height. Default is determined at run time by the Term::ReadKey
module.

=item width

The column width. Default is determined at run time by the Term::ReadKey
module.

=item color_range

Pass an array ref with a range (0 to 255) of color values which will be used
accross each bar. The default is no colors at all.

=item style

Either 'dot' or 'bar'. Default is 'bar'.

=item orientation

Either 'vertical' or 'horizontal'. Default is 'horizontal'.

=item border

0 is no border, 1 to 255 to indicate the color of the border. The default is 0.

=back

=head3 add_value(<hash ref>)

Add a value to be graphed. Accepts a hash ref with:

=over

=item label

The optional text label.

=item value

The numeric value being charted. (If a non-numeric value is supplied, then
its string length will be used as the value.)

=back

=head3 add_values(<list>)

Add a list of values (without labels) all at once.

=head3 print()

This will print the rendered chart to STDOUT.
