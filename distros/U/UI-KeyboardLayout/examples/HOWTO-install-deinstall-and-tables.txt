==================== INSTALL of keybords of http:/k.ilyaz.org

Four "personalities" are available (each accessing 4 scripts, but with 
different “base” scripts):
  iz-la-4s = Latin Script    (with access to Cyrillic, Greek, Hebrew, Math etc)
  iz-ru-4s = Cyrillic Script (with access to Latin, Greek, Hebrew, Math etc)
  iz-gr-p4 = Greek Script    (with access to Latin, Cyrillic, Hebrew, Math etc) 
  				[simplified Polytonic input]
  iz-hb-4s = Hebrew Script    (with access to Latin, Greek, Cyrillic, Math etc)

   [You need to install several only if you want to type long
    stretches of text in two different scripts.  Otherwise one may be
    enough: it is possible to enter short stretches of “foreign”
    scripts without switching a personality.]

For each personality (unzip first, then in the corresponding directory):
     TO INSTALL:   run setup.exe.
     TO UNINSTALL: run setup.exe, confirm "remove"
			(or do it from Control Panel / Remove Program).
 	(To update, do UNINSTALL + INSTALL.  If this keyboard
	 has been used in this session, one needs to log off and
	 relogin…)	

  In both cases, administrative privileges are required.  In particular, at
  a certain moment a hidden window may appear (watch for a blinking
  "task icon" on the Windows panel; click it for window to appear) - one
  will need to confirm changes to the system in this window.

    (Manual install is possible, but tricky.  See
      https://github.com/lelegard/winkbdlayouts#declaring-a-keyboard-layout-on-windows
    The discussion is at
      https://stackoverflow.com/questions/75932692/developed-a-windows-keyboard-layout-dll-but-not-loaded-no-error-message/76030384#76030384
    .)

After you install them, the Language Bar will have an entry (or
entries) for this keyboard (look for a keyboard icon in the Windows
panel (usually at bottom of the screen, to the left of "notification icons"); 
if you activated several languages, you may need to choose a keyboard
separately for every language).  The setting may be separate for every
running program.  One can click the Language Bar to switch
between personalities.  Or: set up to switch the personalities
via hot keypresses: Right click on the Language Bar --> Settings
--> Advanced Key Settings (in Win7).

  You can choose the keyboard(s) available for every language at any time;
  go through Language Bar --> Settings (as above) --> General. Keep in mind
  that when one `Add́s, one adds a “Language + Keyboard” pair; any keyboard may
  be matched with any language (but one may need to choose “Show More” in the
  keyboard selector).

  For many (most?) programs the Language choice would not matter (much?); 
  however, the other programs may change behaviour according to the chosen 
  language; word processor may also STORE the information about the current 
  language as “attributes” of the typed-in text, so that the correct 
  spell-checker would be chosen for every run of the text in mixed-language
  documents.

Unfortunately, the install packages generated by MSKLC are dumbed down
a lot: to uninstall, one needs to exactly match the setup.exe to the
keyboard you want to remove.  Remember: the name of directory of setup.exe 
is identical to the name of the keyboard DLL.  Additionally, one
can match filenames of DLLs and names of keyboards by inspecting entries in

  HKEY_LOCAL_MACHINE\SYSTEM\ControlSet001\Control\Keyboard Layouts

in regedit.  (Keyboards may also be removed via the "Remove program"
section of Control Panel.)

Note that on Windows, a keyboard layout is associated with a DLL name
AND with its description; two layouts can't share these, and I have seen
a system seriously confused when a new keyboard was installed with the
description differing only in a few positions from the old one.  Therefore
I do not supply the version number of the layout in its description.

  (It is in the properties of the DLL file, the last two numbers in the
   version number — in Details — like 1.0.750.750 — match the version
   numbers of the .pm creator and of the klc2c converter.)	

On Windows 8, one may need to reboot before the layout becomes active; compare
with http://blogs.msdn.com/b/michkap/archive/2012/03/12/10281199.aspx

Note that releases before 0.66 need a different deinstaller.  Look in
  http://k.ilyaz.org/versions/v0.65

Remark: the more advanced “shortcuts” for switching between layouts
are buggy on Windows: sometimes this setting is forgotten, sometimes
“extra” shortcuts appear.  Still, the shortcuts usually re-appear
after a reboot, so it is conventient to have them — especially if you
want to also use the standard layouts too.  For example, if you use
Alt-Shift for switching, you may set Alt-Shift-1 to switch to
iz-Latin, and Alt-Shift-0 to switch to std-US.  Likewise, use Alt-Shift-2
for iz-Russian, and Alt-Shift-9 for std-Russian.

  (It is VERY advisable to switch off Ctrl-Shift switching to the next
   keyboard in the same language.  Most people find this very
   annoying. — And after switching this off I never saw any problem of
   those mentioned in the preceding paragraph.)

======================================================= BUILD

The izKeys keyboards are built by
  http://search.cpan.org/~ilyaz/UI-KeyboardLayout/lib/UI/KeyboardLayout.pm
using the KBD description [text] file version 0.77 (the latest version is on
  http://search.cpan.org/~ilyaz/UI-KeyboardLayout/examples/izKeys.kbdd
append version number to "UI-KeyboardLayout" in this URL if needed).  The build recipe is in:
  http://search.cpan.org/~ilyaz/UI-KeyboardLayout/examples/build_here.cmd

======================================================= HEURISTICS

Currently, the most accessible descriptions are those in
  http://k.ilyaz.org/
  http://k.ilyaz.org/windows/izKeys-visual-maps.html
A sketchy description of these keyboards may be extracted from
  http://search.cpan.org/~ilyaz/UI-KeyboardLayout/Changes
A more readable description germinates in
  http://search.cpan.org/~ilyaz/UI-KeyboardLayout/lib/UI/KeyboardLayout/izKeys.pod
(after one have seen the visual maps, the tables in
  http://k.ilyaz.org/windows/coverage-1prefix-Latin.html
  http://k.ilyaz.org/windows/coverage-1prefix-Cyrillic.html
may become more clear).

The .pm file contains some motivations.  Many ideas of this layout were first
honed on the prototype pilot version called EurKey++ which borrowed the Latin
base face and the Greek/Math/Typography layouts of EurKey
  http://eurkey.steffen.bruentjen.eu/layout.html
see
  http://k.ilyaz.org/v0/

Note that the latest release of that distribution can be found in
  http://search.cpan.org/~ilyaz/UI-KeyboardLayout/
(follow the "Latest release" link there — but this is usually not needed).

The following tables are also available [in an enriched form] in
coverage-1prefix-*.html in the same directory as this file.
